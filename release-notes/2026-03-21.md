# Release Notes – 2026-03-21

## Highlights

1. **Level-Based Logger Architecture**  
   - Introduced standard log levels (`error`, `warn`, `info`, `debug`, `trace`) with consistent filtering logic.  
   - `TOOLPACK_SDK_LOG_LEVEL` and `logging.level` now control verbosity everywhere.  
   - Added explicit helpers (`logError`, `logWarn`, `logInfo`, `logDebug`, `logTrace`) used across providers and tool executors.

2. **Removal of Deprecated `TOOLPACK_SDK_LOG_VERBOSE`**  
   - Logger no longer accepts the legacy verbose flag (env or config).  
   - All docs, sample `.env` files, and configs now reference `logging.level` / `TOOLPACK_SDK_LOG_LEVEL` instead.  
   - `logging.verbose` option was deleted from `ToolpackConfig`.

3. **Search Tool Logging Improvements**  
   - `web.search` routes Tavily/Brave/DuckDuckGo diagnostics through the new level APIs.  
   - Added numeric freshness mapping so Tavily receives valid `days` values, restoring “latest results” support across providers.

4. **Configurable Tool Result Truncation**  
   - Introduced `tools.resultMaxChars` in `toolpack.config.json` to cap per-tool output length.  
   - SDK now uses this config (default 20k chars) instead of the `TOOLPACK_SDK_TOOL_RESULT_MAX_CHARS` env var.

5. **Knowledge Module Progress + Docs**  
   - `Knowledge.sync` now counts chunks up front and emits `embedding:start/progress/complete` at 10% milestones.  
   - Updated README + Knowledge guide with the new progress API and configuration reference.

6. **CLI Knowledge Loading UX**  
   - Sample CLI shows a centered loading card driven by `onEmbeddingProgress` and hides the prompt until embeddings finish.  
   - Knowledge assets (Markdown, JSON, SQLite) are bundled under `samples/toolpack-cli/knowledge` and auto-copied into `~/.toolpack/knowledge` at startup for portability.

7. **Knowledge Integration Tests**  
   - Added `tests/integration/knowledge/knowledge-sources.test.ts` covering Markdown, JSON, and SQLite ingestion end-to-end with a deterministic embedder.

8. **Persistent Knowledge Provider + CLI Cache Bundling**  
   - Introduced `PersistentKnowledgeProvider` with `hasStoredChunks()` and a `reSync` flag on `Knowledge.create()` so applications can skip re-embedding when cached chunks exist.  
   - Sample CLI now bundles pre-embedded JSON chunks inside `samples/toolpack-cli/knowledge/cli` and copies them to `~/.toolpack/knowledge/cli` the first time it runs, ensuring global npm installs launch instantly.  
   - Added a build step (`scripts/copy-knowledge.cjs`) plus `package.json` updates so published CLI packages include the knowledge assets.

## Compatibility Notes

- Existing `logging.enabled` / `logging.filePath` settings continue to work.  
- `TOOLPACK_SDK_LOG_FILE` still implies logging enabled; combine with `TOOLPACK_SDK_LOG_LEVEL` for verbosity.  
- `tools.resultMaxChars` controls truncation; the `TOOLPACK_SDK_TOOL_RESULT_MAX_CHARS` env var was removed.  
- Consumers relying on `TOOLPACK_SDK_LOG_VERBOSE` must switch to the level-based controls.
