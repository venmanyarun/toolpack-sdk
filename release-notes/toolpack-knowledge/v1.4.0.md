# Toolpack Knowledge v1.4.0 Release Notes

**Release Date:** April 12, 2026  
**Previous Version:** v1.2.0

## Overview

This release brings major enhancements to the knowledge package including hybrid search capabilities, new content sources (Web URL and API), persistent storage provider, and advanced features like embedding progress tracking and sync event handling.

## New Features

### Hybrid Search
- **Combined semantic + keyword search** for improved retrieval accuracy
- **Configurable weighting** via `semanticWeight` parameter (default: 0.7)
- **Three search modes**:
  - `semantic` - Pure vector similarity search
  - `keyword` - TF-IDF based text matching
  - `hybrid` - Combined approach with weighted scoring
- Keyword extraction and matching utilities in `utils/keyword.ts`

### New Content Sources

#### WebURLSource
- Index content from web pages
- Automatic HTML parsing and text extraction
- URL normalization and deduplication
- Content-type aware processing
- Support for multiple URLs in a single source

#### APISource
- Fetch and index content from REST APIs
- Configurable HTTP headers and authentication
- Custom request methods (GET, POST, etc.)
- JSON response parsing with customizable extractors
- Support for paginated API responses

### Persistent Knowledge Provider
- **Disk-based storage** for knowledge embeddings and metadata
- **Automatic resync detection** via `shouldReSync()` method
- **Persistent state** across application restarts
- Implements the `KnowledgeProvider` interface

### Event Handling & Progress Tracking
- **`onSync` callback** - Monitor sync lifecycle events (`start`, `file`, `chunk`, `complete`, `error`)
- **`onEmbeddingProgress` callback** - Real-time progress tracking during embedding
  - Track current file, processed count, total count, and percentage
- **`onError` handler** - Custom error handling with `skip` or `abort` options
- **`streamingBatchSize`** - Configurable batch processing for memory-efficient embedding

## Improvements

### Query System
- Enhanced `QueryOptions` interface with `searchType` and `semanticWeight`
- Better result scoring with `combineScores()` utility
- Improved filter matching with cosine similarity

### Provider Interface
- Optional `shouldReSync()` method for automatic resync detection
- Better cleanup handling on knowledge base deletion
- Enhanced error propagation

### Sync Process
- Event-driven sync lifecycle
- Per-file and per-chunk progress tracking
- Graceful error handling with configurable recovery strategies

## API Changes

### KnowledgeOptions Interface
```typescript
interface KnowledgeOptions {
  provider: KnowledgeProvider;
  sources: KnowledgeSource[];
  embedder: Embedder;
  description: string;
  reSync?: boolean;
  onError?: ErrorHandler;              // NEW
  onSync?: SyncEventHandler;           // NEW
  onEmbeddingProgress?: EmbeddingProgressHandler;  // NEW
  streamingBatchSize?: number;          // NEW
}
```

### QueryOptions Interface
```typescript
interface QueryOptions {
  limit?: number;
  filter?: Filter;
  searchType?: 'semantic' | 'keyword' | 'hybrid';  // NEW
  semanticWeight?: number;                         // NEW (0.0 - 1.0)
}
```

## Migration Guide

### Using Hybrid Search
```typescript
const results = await knowledge.query('search term', {
    searchType: 'hybrid',
    semanticWeight: 0.7,
    limit: 10,
});
```

### Adding Progress Tracking
```typescript
const kb = await Knowledge.create({
    provider,
    sources,
    embedder,
    description: 'My knowledge base',
    onEmbeddingProgress: (event) => {
        console.log(`${event.source}: ${event.percent}% (${event.current}/${event.total})`);
    },
    onSync: (event) => {
        if (event.type === 'complete') {
            console.log('Sync completed!');
        }
    },
});
```

### Using WebURLSource
```typescript
import { WebURLSource } from 'toolpack-knowledge/sources';

const source = new WebURLSource({
    name: 'documentation',
    urls: [
        'https://docs.example.com/getting-started',
        'https://docs.example.com/api-reference',
    ],
});
```

### Using APISource
```typescript
import { APISource } from 'toolpack-knowledge/sources';

const source = new APISource({
    name: 'api-docs',
    endpoint: 'https://api.example.com/v1/docs',
    headers: {
        'Authorization': 'Bearer token',
    },
    extract: (data) => data.articles,  // Custom extractor
});
```
