# Toolpack SDK v1.4.0 Release Notes

**Release Date:** April 12, 2026  
**Previous Version:** v1.3.0

## Overview

This release introduces powerful new capabilities including Kubernetes tools for container orchestration, Human-in-the-Loop (HITL) support for action tools requiring confirmation, enhanced MCP client functionality, and significant improvements to the knowledge package with hybrid search and new content sources.

## New Features

### Kubernetes Tools (k8s-tools)
- **New tool category** for Kubernetes cluster management and operations
- **11 kubectl-based tools**:
  - `k8s.list_pods` - List pods in a namespace
  - `k8s.describe` - Describe any Kubernetes resource
  - `k8s.get_logs` - Retrieve container logs
  - `k8s.apply_manifest` - Apply YAML manifests to the cluster
  - `k8s.delete_resource` - Delete Kubernetes resources
  - `k8s.list_services` - List services in a namespace
  - `k8s.list_deployments` - List deployments in a namespace
  - `k8s.get_config_map` - Retrieve ConfigMap data
  - `k8s.switch_context` - Switch between kubectl contexts
  - `k8s.get_namespaces` - List all namespaces
  - `k8s.wait_for_deployment` - Wait for deployment rollout to complete
- Full TypeScript support with comprehensive test coverage
- Example usage documentation included

### Human-in-the-Loop (HITL) for Action Tools
- **Configurable confirmation system** for sensitive operations
- **Risk levels**: `high`, `medium` for tool operations
- **Flexible bypass options**:
  - By tool category (e.g., `filesystem`, `database`)
  - By specific tool names (e.g., `fs.delete_file`, `db.delete`)
  - Global enable/disable toggle
- **Per-tool confirmation configuration** via `confirmation` property
- Supports `showArgs` to display relevant arguments in confirmation prompts
- Complete test suite for HITL logic validation

### Enhanced Knowledge Package (toolpack-knowledge v1.4.0)

#### Hybrid Search
- **Combined semantic + keyword search** for improved retrieval accuracy
- **Configurable weighting** (`semanticWeight`) between semantic and keyword scores
- **Three search modes**: `semantic`, `keyword`, `hybrid`
- Keyword matching with TF-IDF scoring on file content

#### New Content Sources
- **API Source** (`APISource`): Fetch and index content from REST APIs
  - Configurable headers, authentication, and request methods
  - JSON response parsing with customizable extractors
  - Support for paginated API responses
- **Web URL Source** (`WebURLSource`): Index web page content
  - Automatic HTML parsing and text extraction
  - URL normalization and deduplication
  - Content-type aware processing

#### Advanced Features
- **Persistent Knowledge Provider**: New provider with disk-based storage
- **Embedding progress tracking**: Real-time progress callbacks during sync
- **Error handling hooks**: Configure `onError` for custom error handling (`skip` or `abort`)
- **Sync event handlers**: Monitor sync lifecycle with `onSync` callbacks
- **Streaming batch processing**: Configurable `streamingBatchSize` for memory-efficient embedding
- **Automatic resync detection**: Provider can trigger automatic resync when needed

### MCP Client Enhancements
- **Improved tool discovery** with better error handling
- **Enhanced server connection management**
- **Schema validation improvements** for MCP tool parameters
- **Better resource management** for persistent connections

## Breaking Changes

### Knowledge Package Interface Updates
- `Knowledge.create()` now accepts `KnowledgeOptions` with additional callbacks
- `QueryOptions` extended with `searchType` and `semanticWeight` parameters
- Providers may now implement optional `shouldReSync()` method

### HITL Configuration
- New `HitlConfig` interface for client initialization
- Tools with `confirmation` property may trigger confirmation prompts
- Action tools now require explicit HITL configuration for automation

## Migration Guide

### Enabling HITL
```typescript
const client = new AIClient({
    providers: { /* ... */ },
    defaultProvider: 'default',
    hitlConfig: {
        enabled: true,
        bypassCategories: ['coding'], // Skip confirmation for coding tools
        bypassTools: ['fs.read_file'], // Skip confirmation for specific tools
    },
});
```

### Configuring Tool Confirmation
```typescript
const deleteTool: ToolDefinition = {
    name: 'custom.delete',
    category: 'filesystem',
    confirmation: {
        level: 'high',
        reason: 'This operation permanently deletes data',
        showArgs: ['path'],
    },
    // ... rest of tool definition
};
```

### Using Hybrid Search
```typescript
const results = await knowledge.query('search term', {
    searchType: 'hybrid',
    semanticWeight: 0.7, // 70% semantic, 30% keyword
    limit: 10,
});
```

### New Knowledge Sources
```typescript
import { Knowledge } from 'toolpack-knowledge';
import { WebURLSource, APISource } from 'toolpack-knowledge/sources';

const kb = await Knowledge.create({
    provider: /* ... */,
    sources: [
        new WebURLSource({
            name: 'docs',
            urls: ['https://docs.example.com'],
        }),
        new APISource({
            name: 'api-docs',
            endpoint: 'https://api.example.com/docs',
            headers: { 'Authorization': 'Bearer token' },
        }),
    ],
    // ...
});
```

### Kubernetes Tools Usage
```typescript
import { k8sToolsProject } from 'toolpack-sdk/k8s-tools';

const toolpack = new ToolpackSDK({
    projects: [k8sToolsProject],
});

// Use in your agent workflow
const result = await toolpack.execute('k8s.list_pods', {
    namespace: 'default',
    context: 'production',
});
```
