# Toolpack SDK v1.3.0 Release Notes

**Release Date:** April 5, 2026  
**Previous Version:** v1.2.0

## Overview
This release introduces major workflow system enhancements including a new CODING_MODE, workflow presets, custom workflow prompts, and several critical workflow engine fixes. The update also includes comprehensive documentation updates across README files and the documentation site.

## New Features

### CODING_MODE - New Built-in Mode
- **Third built-in mode** optimized for software development tasks
- Concise planning prompts with minimal conversational text
- Focused on file operations and code changes
- Pre-configured with coding-specific tools (`coding.read_code`, `coding.search_code`, etc.)
- Ideal for refactoring, debugging, and file manipulation tasks

### Workflow Presets System
- **Three workflow presets** for different use cases:
  - `AGENT_WORKFLOW` - Full planning for complex autonomous tasks
  - `CODING_WORKFLOW` - Concise planning for development tasks
  - `CHAT_WORKFLOW` - Direct execution for conversational responses
- Centralized workflow definitions in `presets.ts`
- Named workflows with `name` field in `WorkflowConfig`
- Custom planning and step prompts for each workflow type

### Custom Workflow Prompts
- **`planningPrompt`** - Override default planning behavior with custom system prompts
- **`stepPrompt`** - Control how individual steps are executed
- Enables fine-tuned workflow behavior for specialized use cases
- Full documentation in new Custom Workflows guide

## Improvements

### Workflow Engine Enhancements
- **Removed redundant `createImplicitPlan` method** - Now directly uses `createPlan`
- **Enhanced dynamic steps context** - Includes remaining plan steps to prevent redundant suggestions
- **Fixed plan header leak** - Removed direct yield into message stream, using events instead
- **Improved step execution prompts** - Better context injection with plan summary and previous results

### Code Quality
- **Removed unimplemented `try_alternative` strategy** - Cleaned up type definitions to prevent silent contract violations
- **Added `message-utils.ts`** - Utility functions for message processing
- **Better query classification** - Enhanced routing logic in `query-classifier.ts`


## Bug Fixes

### Workflow System
- **Fixed test failure** - Updated `planner.test.ts` to use `createPlan` instead of removed `createImplicitPlan`
- **Removed plan header from message stream** - Prevents metadata leakage in planning-only mode
- **Fixed workflow naming** - Consistent `name` field across all workflow configurations


## Breaking Changes

### Workflow API Changes
- **`createImplicitPlan` method removed** from `Planner` class
  - **Migration:** Use `createPlan()` directly instead
  
- **`try_alternative` failure strategy removed** from `WorkflowConfig.onFailure.strategy`
  - **Migration:** Use `'abort'`, `'skip'`, or `'ask_user'` strategies instead

### Type Changes
- **`WorkflowConfig` interface updated**:
  - Added optional `name?: string` field
  - Added `stepPrompt?: string` in `steps` configuration
  - Removed `'try_alternative'` from failure strategy union type
  - Removed `allowAlternativePath` field

## Migration Guide

### Updating to CODING_MODE
```typescript
// New mode available
toolpack.setMode('coding');

// Or use in custom mode configuration
import { CODING_WORKFLOW } from 'toolpack-sdk';

const myMode = createMode({
    name: 'my-coding',
    workflow: CODING_WORKFLOW,
});
```

### Using Workflow Presets
```typescript
// Import workflow presets
import { AGENT_WORKFLOW, CODING_WORKFLOW, CHAT_WORKFLOW } from 'toolpack-sdk';

// Use in mode configuration
const customMode: ModeConfig = {
    name: 'my-mode',
    workflow: AGENT_WORKFLOW,  // or CODING_WORKFLOW, CHAT_WORKFLOW
};
```

### Custom Workflow Prompts
```typescript
const customWorkflow: WorkflowConfig = {
    name: 'research',
    planning: {
        enabled: true,
        planningPrompt: `Your custom planning instructions...`,
    },
    steps: {
        enabled: true,
        stepPrompt: `Your custom step execution instructions...`,
    },
};
```

### Failure Strategy Updates
```typescript
// Before (v1.2.0)
onFailure: {
    strategy: 'try_alternative',  // No longer supported
}

// After (v1.3.0)
onFailure: {
    strategy: 'ask_user',  // Use 'abort', 'skip', or 'ask_user'
}
```
