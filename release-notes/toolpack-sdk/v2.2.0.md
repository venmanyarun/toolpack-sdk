# Toolpack SDK v2.2.0

Date: 2026-06-08

## Highlights

- **MCP Server** — expose Toolpack's 100+ built-in tools to any MCP client (Claude Desktop, Cursor, custom agents) via HTTP or stdio, with static/JWT/custom auth and tool search mode
- **Two new exec tools** — `exec.run_blocking` (no-timeout shell execution) and `exec.tail_output` (read tail of background process output)
- **Kubernetes tools now auto-loaded** — `k8sToolsProject` was missing from `loadBuiltIn()`; all 11 Kubernetes tools are now registered by default (tool count: 95 → 106)

---

## Breaking changes

None.

## New features

### `sdk.startMcpServer()` — Toolpack as an MCP server

```typescript
const handle = await sdk.startMcpServer({
  transport: 'http',   // or 'stdio' for Claude Desktop / Cursor
  port: 3000,
});
// handle.port — actual bound port (useful with port: 0)
// handle.toolCount — number of tools exposed
// handle.stop() — graceful shutdown
```

**Transport options:**
- `'http'` — Streamable HTTP transport for remote/cloud use
- `'stdio'` — for Claude Desktop and Cursor via `claude_desktop_config.json`

**Tool filtering:**
```typescript
expose: { categories: ['filesystem', 'version-control'] }
expose: { tools: ['fs.read_file', 'git.commit'] }
```

**Authentication (HTTP only):**
- `mode: 'static'` — pre-shared bearer tokens; supports multiple tokens for zero-downtime rotation
- `mode: 'jwt'` — JWKS-based verification (Auth0, Supabase, Clerk); auto-mounts `/.well-known/oauth-protected-resource` when `serverUrl` is set
- `mode: 'custom'` — bring your own `verifyAccessToken()` function

**Search mode:**
```typescript
searchMode: true  // tools/list returns only tool.search; clients discover tools on demand
```
Reduces context token usage for large tool sets — clients call `tool.search` first, then the specific tool by name.

**Agent exposure:**
Agents from `@toolpack-sdk/agents` can be exposed as MCP tools via `McpChannel.asAgentDefinition()`. They appear in `tools/list` as `agent.<name>`.

**`port: 0` support:**
Pass `port: 0` to let the OS assign a free port. Read the actual port from `handle.port`.

### New exec tools

| Tool | Description |
|---|---|
| `exec.run_blocking` | Execute a shell command with no timeout — waits until the process exits naturally. Returns exit code, stdout, stderr. |
| `exec.tail_output` | Read the last N lines of output from a background process started with `exec.run_background`. Returns alive status, exit code, and recent stdout/stderr lines. |

### Bug fix: Kubernetes tools missing from `loadBuiltIn()`

`k8sToolsProject` existed in the codebase but was never imported in `loadBuiltIn()`. All 11 Kubernetes tools (`k8s.get_pods`, `k8s.apply_manifest`, `k8s.get_logs`, etc.) are now registered by default when `tools: true` is set. Tool count goes from 95 to 106.
