# Toolpack SDK v2.1.0

Date: 2026-05-28

## Highlights

- **Vertex AI migrated to the new Google Gen AI SDK** — the `vertexai` provider now uses `@google/genai` instead of the deprecated `@google-cloud/vertexai` package (deprecated June 2025, removal June 2026).
- **Automatic rate-limit retry** — provider `generate()` calls now retry transient `429` errors with configurable backoff via the new `withRetry` utility.
- **Per-mode `response_format`** — modes can now force `json_object` output, ideal for evaluator/parser agents whose final response must be machine-readable.
- **Resilient tool execution** — a single tool failure can no longer orphan sibling tool calls; every `tool_call` always receives a matching `tool_result`.

---

## Breaking changes

None. This release is fully backward compatible.

> Note: the `vertexai` provider now depends on `@google/genai` instead of `@google-cloud/vertexai`. The `VertexAIConfig` public API (`projectId`, `location`, `googleAuthOptions`) is unchanged, so no consumer code changes are required.

---

## New features

### Per-mode response format

Modes accept an optional `response_format` to constrain text output:

```typescript
import { createMode } from 'toolpack-sdk';

const evaluatorMode = createMode({
  name: 'evaluator',
  response_format: 'json_object', // 'text' (default) | 'json_object'
});
```

- Applied to `generate()`, follow-up, and `stream()` requests for the active mode.
- Tool-call rounds are unaffected — the model still returns `functionCall` parts normally; the format only governs text content.

### Rate-limit retry

A new `withRetry` utility (`utils/retry.ts`) retries transient failures:

- Retries `RateLimitError` (HTTP 429) up to 3 times by default.
- Honors a provider-supplied `retryAfter` value, falling back to a `[10s, 30s, 60s]` backoff schedule.
- Configurable via `maxRetries`, `backoffMs`, `isRetryable`, and an `onRetry` hook for logging/instrumentation.
- Wired into all provider `generate()` calls in the client.

## Improvements

- **Vertex AI provider** — rewritten to use `GoogleGenAI({ vertexai: true })` with `models.generateContent` / `models.generateContentStream`, replacing the deprecated `getGenerativeModel` + `startChat()` flow.
- **`tool-orchestrator.ts`** — `executeBatchWithLimit` now uses `Promise.allSettled`, so a single tool failure never drops sibling results. Each failed call returns a structured error entry instead of rejecting the batch.
- **`client/index.ts`** — both parallel and sequential tool-execution paths are individually guarded so every `tool_call` in an assistant turn always gets a matching `tool_result`, preventing orphaned tool calls (e.g. on circular dependencies or executor throws).

---

## Fixes

- **Gemini & Vertex AI JSON mode** — `responseMimeType: 'application/json'` is now applied only when no tools are present. Gemini/Vertex do not support JSON mode and function declarations simultaneously and would otherwise truncate the response. JSON mode is honored for tool-free requests.
